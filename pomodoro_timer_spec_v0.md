# プロジェクト仕様書: ポモドーロタイマー（v0.6）

---

## プロジェクト名

**ESP32C3 Pomodoro Timer**

---

## プロジェクトの背景

在宅・個人開発など、長時間作業で集中と休憩のバランスを取りたい場面が多く、  
スマホアプリでは気が散りやすいため「**デスク常設型・物理ボタン操作のポモドーロタイマー**」を構築する。

目的は、

- **視覚的に進捗がわかる UI（TFT 表示）**
- **シンプルな操作（3 ボタン）**
- **集中時間の自動記録（microSD CSV）**  
  を両立すること。

初期版は最小限構成で、Wi-Fi 同期とデータ保存を確立。  
将来的には BGM 再生・バッテリー駆動・Web 連携を拡張予定。

---

## プロジェクト概要

- **マイコン**：Seeed Studio XIAO ESP32-C3
- **ディスプレイ**：ST7789 240×240 TFT（SPI）
- **入力**：3 ボタン（アナログラダー方式）
- **記録**：microSD（SPI）に CSV 出力
- **音**：圧電ブザー（受動型）
- **時刻同期**：Wi-Fi 経由 NTP（RTC なし）
- **電源**：USB 給電（試作時）
- **形式**：デスク据置タイプ（名刺サイズ）

### 動作仕様

#### ■ サイクル

- 50 分作業 → 5 分休憩 → 50 分作業 → 10 分休憩（固定プリセット）
- 自動遷移、Next/Prev 操作あり

#### ■ ボタン構成（アナログラダー 1 ピン）

| ボタン   | 操作         | 機能                         |
| -------- | ------------ | ---------------------------- |
| ボタン 1 | 短押し       | Start / Pause                |
|          | 長押し 2 秒  | Next（スキップ）             |
|          | 長押し 4 秒  | Prev（戻る）                 |
|          | 長押し 10 秒 | リセット（保存データ初期化） |
| ボタン 2 | シングル     | −1 分                        |
|          | ダブル       | 音量 DOWN                    |
| ボタン 3 | シングル     | +1 分                        |
|          | ダブル       | 音量 UP（0〜10 段階）        |

※ ボタン 1 を押し続けている間は、経過秒ごとに短いビープ音を鳴らし、長押し時間の目安をフィードバックする。

#### ■ ディスプレイ表示

- リング進捗表示（240×240 円形 UI）
- 残り時間（mm:ss → h:mm:ss 自動拡張）
- 今日のトータル集中時間
- 状態に応じて「PAUSED」表示、アラート時は点滅カウント表示

#### ■ 音

- 圧電ブザーによるトーン出力
- 1 分前 1 回、終了 5〜1 秒は毎秒ビープ
- 音量 0〜10 段階（ON 時間で制御）
- Wi-Fi 接続成功時は音量レベル 1 で 1 回のみビープ通知
- Wi-Fi 接続失敗／NTP 同期失敗時は音量レベル 1 で 2 回ビープ通知

#### ■ 時刻同期

- 起動時および 1 時間ごとに NTP 同期
- 同期失敗時は内部継続、成功時に補正
- 最終同期 epoch を保存して再開時に復元

#### ■ 電源断対応

- LittleFS に `/system/checkpoint.json` を 60 秒ごと保存
- 再起動時に自動復元（Pause 状態も保持）
- CSV はスライス完了時のみ書込（中間保存なし）

#### ■ CSV 保存

- ファイル名：`/logs/work.csv`（1 ファイル継続追記）
- 各スライス終了（Pause/Reset/Skip/Prev/Completed）時に即追記
- **day_cutover 有効**：日付を跨ぐ際に区切り行を追加して再スタート

#### CSV カラム仕様

```
session_id, segment_id, start_timestamp, end_timestamp, duration_sec, end_reason, volume_level, cycle_label
```

end_reason:

- completed / paused / reset / skipped / prev / day_cutover / recovered_drop

#### 出力例

```
1,1,2025-10-03 09:00:00,2025-10-03 09:50:00,3000,completed,7,Work#1
2,1,2025-10-03 10:00:00,2025-10-03 10:10:15,615,paused,6,Work#1
2,2,2025-10-03 10:12:00,2025-10-03 10:50:00,2280,completed,6,Work#1
3,1,2025-10-03 23:40:00,2025-10-03 23:59:59,1199,day_cutover,6,Work#2
4,1,2025-10-04 00:00:00,2025-10-04 00:10:00,600,completed,6,Work#2
5,1,2025-10-04 14:15:20,2025-10-04 14:22:18,418,recovered_drop,5,Work#1
```

---

### 時間処理

- 内部 tick: 10ms
- UI 更新: 250ms 間隔
- アラート: 残り 60s および 5〜1s
- ±1 分調整：下限 0 分、上限なし（h:mm:ss 対応）

---

### アナログラダー仕様

| 押下 | 抵抗値 | ADC 範囲(12bit) |
| ---- | ------ | --------------- |
| BTN1 | 10kΩ   | <840            |
| BTN2 | 47kΩ   | 840–1480        |
| BTN3 | 68kΩ   | 1480–2850       |
| なし | —      | ≥2850           |

デバウンス: 20ms  
ダブルタップ: 300ms  
長押し: 2s / 4s

※ 12bit ADC（0〜4095）、3.3V 系、100kΩ プルアップを前提にした理論値。実機ではキャリブレーション用定数で微調整する。

---

### エラーハンドリング

- SD エラー: 「SD!」表示、USB-CDC へ即時ログ出力
- Wi-Fi エラー: 「OFFLINE」表示、内部時刻継続
- 書込リトライ: 3 回まで試行
- 異常終了: recovered_drop 行で記録

## CSV カラム仕様詳細

| カラム名            | データ型                        | 内容                                                               | 発生タイミング・補足                                                                                                                                                                                                                                                                                                                                                |
| ------------------- | ------------------------------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **session_id**      | 整数（連番）                    | 同一日内の作業セッション（1 回の「work」区間）を識別する ID。      | - 1 日で複数回の作業がある場合、work 開始時にインクリメント。<br>- 日付が変わるとリセット（01 から再カウント）。<br>- 例：午前・午後の 2 回作業なら 1, 2 になる。                                                                                                                                                                                                   |
| **segment_id**      | 整数（連番）                    | 同一セッション内で Pause や Reset などにより分割された部分を識別。 | - Resume で再開するたびに +1。<br>- 1 つの session_id に複数 segment_id が存在する可能性あり。                                                                                                                                                                                                                                                                      |
| **start_timestamp** | 文字列（`YYYY-MM-DD hh:mm:ss`） | スライスの開始日時（ローカル時刻）。                               | - Start または Resume 時に記録。<br>- NTP 同期済み時刻を使用。同期エラー時は内部クロック。                                                                                                                                                                                                                                                                          |
| **end_timestamp**   | 文字列（`YYYY-MM-DD hh:mm:ss`） | スライスの終了日時（ローカル時刻）。                               | - Pause / Reset / Skip / Prev / Completed / day_cutover など終了イベント発生時。                                                                                                                                                                                                                                                                                    |
| **duration_sec**    | 整数（秒）                      | スライスの継続時間（end - start）。                                | - 秒単位で記録（小数切り捨て）。<br>- CSV 追記直前に計算。                                                                                                                                                                                                                                                                                                          |
| **end_reason**      | 文字列（enum）                  | スライス終了の理由。                                               | 以下の 7 種類（詳細下記）。<br>- `completed`：タイマー完了（残り 0 秒）。<br>- `paused`：ユーザーが Pause 押下。<br>- `reset`：ユーザーが Reset 押下（または強制中断）。<br>- `skipped`：Next 押下でスキップ。<br>- `prev`：Prev 押下で前サイクルに戻る。<br>- `day_cutover`：日付を跨いだため自動分割。<br>- `recovered_drop`：電源断など異常終了 → 復旧時に補完。 |
| **volume_level**    | 整数（0〜10）                   | スライス終了時点での音量設定。                                     | - 音量ボタン操作でリアルタイム更新。<br>- 終了時点の値を出力（0=無音）。                                                                                                                                                                                                                                                                                            |
| **cycle_label**     | 文字列                          | 作業サイクルの識別名（例：`Work#1`, `Work#2`, `Break#1`）。        | - 1 サイクル目、2 サイクル目などの管理に使用。<br>- 自動で連番を付与。                                                                                                                                                                                                                                                                                              |

---

## end_reason の詳細定義

| 値                 | 意味                             | 典型的な発生タイミング     | 備考                               |
| ------------------ | -------------------------------- | -------------------------- | ---------------------------------- |
| **completed**      | タイマーが正常終了（0 秒到達）   | 作業/休憩いずれも          | 次サイクルへ自動遷移               |
| **paused**         | 一時停止（Stop ボタン押下）      | ユーザー操作時             | Resume 時に新 segment 開始         |
| **reset**          | 作業中断・全リセット             | 長押しや異常状態           | セッション再開時は新 session_id    |
| **skipped**        | スキップ操作で次フェーズへ       | 長押し 2 秒                | 現セッションを未完了として終了     |
| **prev**           | 戻る操作で前フェーズへ           | 長押し 4 秒                | 前サイクル再実行時など             |
| **day_cutover**    | 日付を跨いだため分割             | 23:59:59→00:00:00 跨ぎ     | 同一 session を 2 行で表現         |
| **recovered_drop** | 電源断や強制リセット後の復帰検出 | checkpoint.json から再開時 | 失われたスライスの“断片”を明示記録 |

## 解析時の利用例

| 分析目的               | 使用カラム                                    | 説明                               |
| ---------------------- | --------------------------------------------- | ---------------------------------- |
| 1 日の総集中時間       | `duration_sec`（where `end_reason` != break） | 日付別で合計                       |
| 作業の中断頻度         | `end_reason = paused` の件数                  | 集中度合いの推定                   |
| 日を跨ぐ連続作業検出   | `end_reason = day_cutover` の行               | 複数日にまたがる集中セッション確認 |
| 電源障害や異常終了検知 | `end_reason = recovered_drop`                 | 障害発生時刻のトラッキング         |
